# Каталоги и секреты

Секрет — это значение, которое не должно храниться в открытом виде в чатах, документах или репозиториях. Это может быть пароль, токен, приватный ключ, строка подключения или другой чувствительный текст.

## Каталоги

Каталоги помогают навести порядок. Хорошая структура делает секреты понятными без лишних пояснений.

- Разделяйте окружения: `Production`, `Staging`, `Development`.
- Разделяйте назначение: `Базы данных`, `Мониторинг`, `CI/CD`.
- Не усложняйте без необходимости структуру, иначе команде будет трудно искать секреты.

## Имена секретов

Имя должно объяснять назначение без раскрытия самого значения. Например, `Production PostgreSQL password` понятнее, чем `password1`.

## Раскрытие значения

Значение секрета скрыто, пока вы явно не нажмёте на кнопку для раскрытия значения. Если для секрета включено одобрение, сначала будет отправлен запрос владельцу секрета.

## Обновление и удаление

Значения статических секретов можно обновлять вручную, нажав "Заменить значение" на странице секрета. Секреты без шаблонов можно заменить произвольным значением, а шаблонные — только повторно сгенерировать автоматически.

После обновления значения у секрета повысится номер версии, а в истории появится информация о дате обновления.
