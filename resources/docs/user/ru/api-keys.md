# API-ключи

API-ключи можно использовать для автоматизации: CI/CD, внутренних инструментов, скриптов и сервисов, которым требуется получать или обновлять данные без входа пользователя через браузер.

## Когда создавать API-ключ

- Скрипту нужно читать секрет при развёртывании
- Приложение должно создавать или обновлять записи по определённому алгоритму

## Как обращаться с ключом

API-ключ сам является секретом. Его нельзя хранить в открытом репозитории, отправлять в чат или вставлять в общедоступные настройки.

## Ограничивайте права

Выдавайте API-ключу только те права, которые нужны для задачи. Если ключ должен читать один каталог, не давайте ему доступ ко всей организации.

> [!NOTE]
> Если API-ключ больше не нужен или мог быть раскрыт, отзовите его и создайте новый.
